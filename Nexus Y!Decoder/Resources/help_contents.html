Welcome to Nexus Yahoo! Decoder (Beta)

Here i'll talk about some stuff you should maybe know before using this program.

Since there are some programs for decoding and reading Yahoo! Messenger archives offline, i decided to make another example and put it FREE on the net. i may decide to put the source code as open source later. well the program is easy to use you just need to remember some simple rules.

First of all, you have to specify the root directory for the program, in order to read available profiles, botts and YM archives and then to decode them. if you specify an invalid path, you won't be able to do the decoding. we're assuming that your Yahoo! Messenger is installed at the default directory path, which will be something like this:

"C:\Program Files\Yahoo!\Messenger\Profiles\"

Thus you can specify the root directory by clicking the Browse buton at top of the form which will show you a dialog box to select your directory.

Second of all, you have to notice that while decoding an archive file, you cannot open another archive file for decoding or trying to save one. you'll be informed by MessageBoxes from the program according to your actions. also you can see the decoding process in percentage at the bottom of the screen, in a status bar. You can copy the selected text to memory clipboard and you can re-load the selected archive file by clicking the Refresh button in the main toolbar.

Another thing is that you can save your conversations in three formats, HTML Documents, Text Documents and RTF Documents. to do so, first select an archive file to decode, then you can choose File\Save or click on the Save button on the main toolbar. then a dialog box appears, and asks you for the file name and file type. select your desired file format and file name and just click save.

Now let's talk some about YM Decoded Archives:

well, i've seen some algorithms in attempt to decode YM archives. the point is YM archives are not encrypted, rather they have just been XOR'd with your yahoo ID. and then saved in *.dat file format. some of these algorithms (which you can find on the net) will not work properly. i'll give you a simple hint on coded archive files, and im sure wise programmers might find out how to decode these files by reading this.

Well condiser im having a conversation on Yahoo! Messenger with my yahoo! id : "inexuscore" with one friend with his yahoo! id : "james_crack001". yahoo! messenger will simply start a loop through my conversation text and simply XOR every single character with a single character from my yahoo! id. consider this example:

Yahoo! ID: inexuscore

Text: Hello!

The loop will start from "i" which is the first character of my yahoo! id im having a conversation with, and the first character of my conversation text is "H". so these two will be XOR'd together and the result will be saved in a *.dat file. and this loops go's on 'till the end of conversation text. but just notice that if it reaches the last character of my yahoo! id (inexuscore), which is "e", it will start again from the first character and continue the XOR'ing process. You may also have to know that *.dat files are text-based files so reading it's contents will be easy.

Another important thing is how to decode the Date-Time from coded archives ?

Well i've seen some attempts over internet, but here's my tip. if you just open a coded archive file as binary, and then look at the first three bytes of it's contents, you will notice that these three bytes are where the Date-Time data is stored. and the decoding process is the same as i've mentioned above.
